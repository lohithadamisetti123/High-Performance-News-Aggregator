# Performance Audit – High-Performance News Aggregator

## Baseline (Slow Version)

> Baseline measurements taken using Lighthouse (Performance tab) and Chrome DevTools Performance panel for the **slow-version** branch.

| Metric / Issue   | Baseline Score / Observation | Root Cause Analysis | Proposed Solution Hypothesis |
|------------------|-----------------------------|---------------------|------------------------------|
| LCP              | e.g., 8.5s                  | Large, unoptimized hero image blocking render and no dimensions. | Compress hero image, serve WebP, add `width` and `height`, use `srcset`, and appropriate loading behavior. |
| INP (from TBT)   | e.g., TBT: 1200ms, lag on filter input | Re-rendering 500+ DOM nodes on every keystroke; expensive date formatting in render. | Implement list virtualization and memoize expensive computations. |
| CLS              | e.g., 0.45                  | Hero image without intrinsic dimensions causing layout shifts. | Add explicit `width` and `height` attributes for hero image. |
| Bundle Size      | e.g., 1.5MB main bundle     | Full lodash import, no code splitting. | Use cherry-picked lodash imports and implement code splitting for non-critical components. |
| Network Waterfall| 501 serial requests         | Sequential `fetch` calls for 500 items in a loop. | Parallelize story detail requests using `Promise.all`. |
